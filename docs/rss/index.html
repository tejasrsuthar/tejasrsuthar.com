Preparation for AWS certification is very long and time consuming process. Especially if you're already spending your time in a day job. That is mostly the case.

Main AWS Training & Certification Site

In order to get the up to date information you need to visit the main certification and training site located here.

Here you can get most of the details related to the certification like free digital training, classroom training and AWS Certifications.

In my case, I was preparing for AWS Certified Developer - Associate level. This is course specific page where you can get the certificate related details.

In this specific certificate page you'll get information about

  1. Format
    Multiple choice, multiple answer
  2. Type
    Associate
  3. Delivery Method
    Testing center or online proctored exam
  4. Time
    130 mins to complete the exam
  5. Cost
    150 USD (Practice exam: 20 USD)
  6. Language
    Available in English, Japanese, Korean, and Simplified Chinese

Resources

Courses

While preparing for my AWS exam, I referred following Udemy certification course from the author Stephene Maarek.

The author Stephene Maarek is a very good instructor. It helped me a lot. Also apart from this course. And the best signs of a great instructors is that they make learning complex problems so easier that it can be digest very easily for a novice/beginner user as well. Stephene has the skills and ability to do that consistently. And I would highly recommend his course before any second thoughts.

AWS White papers (optional)

I did not explore white papers for my preparation. Though they are very much important for the detailed understanding. They can expand your knowledge of the cloud with AWS technical content authored by AWS and the AWS community

AWS Service FAQs

I considered this FAQs resource for each AWS service a must read section. It gives very very good overview of the each service offering and how to use them. I rad them multiple times for each AWS services. And it really helped me a lot. Most of the questions I got was from FAQs sections.

Strategy

My simple strategy was to just explore the Stephene Maarek's course and and keep notes. I prepared a notes for each AWS Services. The critical part of the certification is that there are so many services you need to learn and its really difficult to keep track of each after hands-on experience.

So its always better to keep a single line notes for each service. I got to create a notebook with stripes for each services marked. And later on when I was preparing on my final days, I just went through each AWS service notes within almost an hour.

Test your skills

Before giving final exam, I would recommend to give practice test exams. This would give you a lot more confidence and you'll do better in your final exams.

For testing exams, I took another course by Stephene Maarek. There were 4 practice tests in a course I took.

Exam Day

On the exam day, I just got enough sleep and prepare from the note I've had created while learning from the course. It took really about an hour to get most of the information for AWS Services.

The exam needs to be given at AWS recognized centers. And I would recommend to reach exam center before an hour just to make sure you get enough time incase if uneven things happens.

Results

AWS Certification exam results are instant. Means you'll get the information about whether you've passed the exam or not.

After a week, you'll be having the result marks in your registered emails.

Show off

And after passing the exam, it give you really great feeling to achieve something, because believe me, this exam is not easy to clear. You've to have enough concentration to answer the questions. And the funny things is that initially all the answers looks same to you. And you'll only notice the difference when you read them multiple times.

Yes, you'll get your badge from AWS Certification account and you can have their certification logo. Also you can add them to your email signature. It looks great.

At the time of writing, the AWS Certification verification site is moved to YourAcclaim Service

Here is my certification badge url for example.

Get better carrier opportunities

After getting the AWS Certification, I believe you'll be having more exposure to potential clients. As clients are specifically looking for the developers who have proved themselves by giving exams as compare to the developers without the certification.

Obviously you should not brag yourself with your certifications but you can certainly consider an edge over your competitor developers in interview and you can have added advantage.

Conclusion

AWS certification is really good recognization and it can open a lot of opportunities to work with potential clients. It really improves the overall AWS services knowledge.

I was really in a need of tech stack where I can write technical blog but it was really time consuming to run a Wordpress blog fore multiple reasons like a it was really heavy for normal tech blogger like me - JamStack with Ghost and github pages.  As I don't want extra clutter to maintain a CMS site. I was always wanted a simple blogging system that is flexible enough that I don't have to spend too much time to maintain it.

Initially I was using a static site generator like Jekyll. It was really good site generator from markdown files. I worked on it for few months and developed a site as well from it and host it to github. After few time I was busy working on other freelancing projects and really didn't look at it.

After couple of years, when I look at the static site generator era, its completely different trend here. Everyone is talking about JamStack and static site generator and there also Ghost CMS was available. It's the best headless CMS I've ever worked with. Typing on its editor is just a breeze.

What is JamStack:
A modern architecture — Create fast and secure sites and dynamic apps with JavaScript, APIs, and prerendered Markup, served without web servers.

What is HeadLess CMS:
A headless content management system, or headless CMS, is a back-end only content management system (CMS) built from the ground up as a content repository that makes content accessible via a RESTful API for display on any device. - WikiPedia

Actually I was looking for a blog solution with following features.

  • minimum maintenance overhead on servers
  • medium cost of handling blog
  • clean editor for typing articles
  • quick and reliable infrastrucutre
  • custom domain setup
  • easy CI / CD support

Each of the above requirements will be resolved by the JamStack hosted to Github Pages.

What I did was,

  • just apply for the custom domain from Godaddy.
  • setup Ghost CMS on local for article writing
  • use wget to generate the static site in /docs folder
  • setup Github Repository and push the code to it
  • setup Github pages published from /docs folder & received domain https://tejasrsuthar.github.io/blog/
  • add CNAME file to /docs folder and provided the custom domain (note: I've not yet setup my custom domain. Just running the blog under the Github Pages domain.)

Overall my experience with this setup for blog is great and I'm planning to write more in-depth articles.

Currently you're reading the blog which is hosted with Github Pages on JamStack.

Conclusion

Now a days, there are so many options to host a static site and I find this way more clinical and suits my requirements. You can still host this static site in S3 bucket as a static site hosting. You need to pay minimal amount to host files there. And with that S3 bucket setup you also have the liberty to use the CloudFront CDN, that will improve caching ability and users get low latency.
